# PROJECT_BRIEF.md

## Basic Information

- **Project Name**: amp-rs
- **Project Type**: api
- **Primary Goal**: Reference implementation of Agent Memory Protocol (AMP) — a local-first MCP server with checkpoints, lessons, semantic search, and code indexing, built in Rust.
- **Target Users**: AI-assisted developers using Claude Code, Cursor, Aider, Cline, or Windsurf, Teams wanting persistent agent memory across sessions, Anyone implementing or evaluating the AMP specification
- **Timeline**: 4 weeks
- **Team Size**: 1

## Functional Requirements

### Key Features (MVP)

- MCP server over stdio transport (standard MCP, works everywhere out of the box)
- Semantic code search — natural language queries across indexed repositories using all-MiniLM-L6-v2 ONNX embeddings and sqlite-vec
- Lessons learned system — store, search, list, and delete engineering lessons with tags and severity levels
- Agent checkpoints — save and restore AI agent working state, track what agents are working on, support multiple agents
- Repository indexing — watch directories for file changes, incremental indexing (mtime-based diff), .gitignore-aware, language-aware chunking
- Checkpoint search — semantic search across saved checkpoints
- Agent status tracking — quick status for any agent (idle/in_progress, current task, checkpoint count)
- On-demand indexing tools — index_repo, diff_index, full_reindex for manual control
- Server status and stats endpoint
- SQLite storage with WAL mode — single-file database, easy backup, embedded everything
- ONNX Runtime embeddings on dedicated thread pool (off async runtime)
- Configurable via CLI flags and config file
- Single binary deployment — no external dependencies, no Python, no cloud APIs
- Health check and basic REST endpoints via Axum

### Nice-to-Have Features (v2)

- In-memory knowledge graph (petgraph) with SQLite persistence — entity types (agent, tool, problem, solution, concept, person, project, chunk), relationship types (used, solved, failed_for, knows, prefers, depends_on, related_to, derived_from), confidence scoring with decay and reinforcement
- Enriched tool schemas — optional graph fields on add_lesson (solved_problem, used_tools, related_concepts) and add_checkpoint (tools_used, problems_encountered, solutions_found, outcome)
- Hybrid search — vector similarity finds entry points, graph traversal expands context with related problems, solutions, tools, concepts
- Graph query API — direct graph traversal exposed as MCP tool (filter by entity type, relationship, direction, confidence threshold)
- Bootstrap tool — seed graph from existing lessons and checkpoints
- Fuzzy entity matching — normalize labels, exact/substring/Levenshtein matching to resolve natural language to graph nodes
- Confidence and integrity system — provisional edges start at 0.3, reinforced by success outcomes, weakened by failures, daily decay with configurable half-life, garbage collection of stale edges
- Web dashboard UI — stats overview, real-time indexing feed, search interface, lesson management (htmx, embedded in binary)
- Prometheus metrics endpoint
- API key authentication

## Technical Constraints

### Must Use

- Rust
- rmcp (official Anthropic MCP SDK for Rust)
- tokio (async runtime)
- rusqlite + sqlite-vec (vector storage)
- ort / ONNX Runtime (embeddings via all-MiniLM-L6-v2)
- axum (HTTP server)
- notify (file watching)
- petgraph (graph, v2 only)
- serde (serialization)
- clap (CLI)

### Cannot Use

- Python
- Cloud APIs
- External databases (must be embedded SQLite only)

## Other Constraints

- Must be a clean reference implementation — no personal workflow assumptions, sensible defaults for any user
- stdio transport for MCP (standard, works with all MCP clients out of the box) — HTTP/SSE transport is acceptable as secondary
- Air-gap compatible — no network calls required for core functionality
- Must run on Linux x86_64 and ARM64 plus macOS
- Single binary + single data file deployment
- All v1 features must work without the graph module (graph is v2, opt-in)
- Embedding model bundled or auto-downloaded on first run
- MIT licensed
