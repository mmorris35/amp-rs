# amp-rs Verification Agent

## Purpose

Validate the completed **amp-rs** application against PROJECT_BRIEF.md. Unlike the executor that checks off deliverables, this agent tries to **break the application** and find gaps.

## Project Context

**Project**: amp-rs — Reference implementation of Agent Memory Protocol (AMP)
**Type**: MCP server (stdio) with embedded SQLite, ONNX embeddings, file watcher
**Language**: Rust

## Verification Checklist

### 1. Smoke Tests
- [ ] `cargo build --release` succeeds
- [ ] `./target/release/amp-rs --help` shows usage
- [ ] `./target/release/amp-rs --version` shows version
- [ ] `./target/release/amp-rs serve` starts without crash (test with timeout)
- [ ] Health check endpoint responds on configured port

### 2. Feature Verification (from PROJECT_BRIEF.md)
For EACH MVP feature, verify it works:

- [ ] **MCP stdio transport**: Server responds to MCP protocol messages
- [ ] **Semantic code search**: Index a repo, search by natural language, get relevant results
- [ ] **Lessons system**: add_lesson, search_lessons, list_lessons, delete_lesson all work
- [ ] **Agent checkpoints**: add_checkpoint, get_recent_checkpoints, search_checkpoints work
- [ ] **Agent status**: get_agent_status returns idle/in_progress correctly
- [ ] **Repository indexing**: index_repo scans files, respects .gitignore
- [ ] **Incremental indexing**: diff_index only re-indexes changed files
- [ ] **Full reindex**: full_reindex clears and rebuilds
- [ ] **Server status**: get_status returns stats
- [ ] **SQLite WAL mode**: Database uses WAL (check with PRAGMA)
- [ ] **Config file**: config.toml is loaded and applied
- [ ] **CLI flags**: --data-dir, --port, --watch-dirs work

### 3. Edge Case Testing
- [ ] Empty database (no lessons, no checkpoints, no indexed code)
- [ ] Very large lesson content (10KB+)
- [ ] Unicode in lesson titles and content
- [ ] Non-existent repo path for index_repo
- [ ] Permission-denied directory for indexing
- [ ] Binary files in indexed repo (should skip)
- [ ] Concurrent checkpoint saves from different agents
- [ ] Database locked by another process

### 4. Error Handling
- [ ] Invalid MCP JSON produces helpful error, not crash
- [ ] Missing required fields return clear error messages
- [ ] Server recovers from non-fatal errors
- [ ] Graceful shutdown on SIGTERM/SIGINT

### 5. Non-Functional
- [ ] `cargo clippy --workspace -- -D warnings` passes
- [ ] `cargo fmt --check` passes
- [ ] `cargo test --workspace` all pass
- [ ] Single binary (no runtime dependencies)
- [ ] README has installation + usage + architecture
